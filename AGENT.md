# SnapRoute ‚Äî Developer Guide

## What This Is

iOS app that registers as a default browser. When a user taps any link, SnapRoute opens instead of Safari and shows quick actions: open in Safari, send to ShelfRead, save to Obsidian. A web preview loads in the bottom half.

## Tech Stack

- Pure SwiftUI, iOS 16+, no external dependencies
- xcodegen for project generation (`project.yml` ‚Üí `SnapRoute.xcodeproj`)
- WKWebView for the link preview (UIViewRepresentable wrapper)

## Default Browser Registration

The app registers as a browser via `Info.plist`:
- `CFBundleDocumentTypes` with `LSItemContentTypes: [public.url, public.html]`
- `LSHandlerRank: Default`

User must manually set SnapRoute as default browser in iOS Settings > Apps > Default Browser App.

## URL Handling Flow

1. User taps a link anywhere in iOS
2. iOS opens SnapRoute (if set as default browser) with the URL
3. `onOpenURL` in `SnapRouteApp` passes URL to `URLRouter`
4. `ActionView` renders: URL bar, 3 action buttons, web preview
5. User picks an action

## Actions

### Open in Safari
Calls `UIApplication.shared.open(url)` ‚Äî opens the URL in Safari.

### Send to ShelfRead
POSTs JSON to the configured ShelfRead ingest URL:
```json
{"from": "snaproute@local", "subject": "Page Title", "htmlBody": "<a href='...'>...</a>"}
```
Note: This sends the URL, not the page content. ShelfRead would need to fetch and process it.

### Save to Obsidian
Opens `obsidian://new` URL scheme with:
- `name`: page title (from WKWebView)
- `content`: markdown with the URL and timestamp
- `vault`: configured vault name
- `path`: configured folder + title

## Settings

Stored in UserDefaults as JSON. Fields:
- `shelfReadIngestURL` ‚Äî Convex ingest endpoint URL
- `obsidianVault` ‚Äî Obsidian vault name
- `obsidianFolder` ‚Äî folder within vault (default: "Inbox")

## Building & Testing

```bash
xcodegen generate                    # regenerate xcodeproj from project.yml
xcodebuild -scheme SnapRoute \
  -sdk iphonesimulator \
  -destination 'platform=iOS Simulator,name=iPhone 17 Pro' \
  build                              # build for simulator
```

Testing URL handling on simulator requires manually setting SnapRoute as default browser in the simulator's Settings app, or using the debug "Test with example.com" button (DEBUG builds only).

## Gotchas

- `xcrun simctl openurl` always goes to Safari ‚Äî can't test default browser behavior via CLI
- `xcrun simctl set-default-browser` doesn't exist ‚Äî must set manually in Settings app
- Info.plist gets overwritten by xcodegen ‚Äî browser entitlements are preserved in project.yml `info.properties` but custom keys like `CFBundleDocumentTypes` need to be in the plist file directly (xcodegen preserves existing plist content when `info.path` is set)
- `LSApplicationQueriesSchemes` must include `obsidian` for `canOpenURL` to work
